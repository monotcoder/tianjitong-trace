# h5-trace — 追溯查询页 PRD

> 技术栈：Vue3 + Vite  
> 域名：`trace.tianjitong.com`（默认）/ 租户自定义域名【VIP】  
> 访问方式：消费者扫描产品标签上的二维码跳转  
> SEO：不需要  

---

## 一、页面结构总览

```
trace.tianjitong.com
├── /:traceCode      （追溯详情页 —— 批次级，核心页面）
├── /label/:code     （电子标签落地页 —— SKU 级扫码介绍，新增）
└── /404             （未找到页面）
```

> 两条路径互补：`/:traceCode` 展示某**批次**的追溯链路；`/label/:code` 展示某 **SKU** 的产品介绍（承诺 / 检测 / 营销 / 自定义）。路由定义时 `/label/:code` 必须在 `/:traceCode` 捕获全路径之前。

## 二、追溯详情页

### 2.1 访问逻辑

```
消费者扫描二维码
       │
       ▼
trace.tianjitong.com/:traceCode
       │
       ├── 查找对应 SKU + 批次信息
       │
       ├── 批次状态正常 ──► 展示追溯详情
       │
       └── 批次已下架/过期 ──► 展示追溯详情 + 顶部红色警告横幅
```

- 每次访问记录扫码日志（时间、IP、设备类型、IP定位城市）
- 无需登录，完全公开访问

### 2.2 下架警告

批次状态为"已下架"或"已过期"时，页面顶部显示：

```
┌─────────────────────────────────────┐
│ ⚠️ 该产品已下架/召回，请勿食用       │
│    如有疑问请联系生产者              │
└─────────────────────────────────────┘
```

红色背景，醒目展示，不可关闭。

## 三、页面信息展示

### 3.1 展示规则

- **优先展示主要基本信息**（固定在顶部，始终展开）
- **其余信息按区块折叠展示**，租户可在后台配置每个区块默认展开或折叠
- **区块顺序**按租户在后台配置的拖拽顺序展示

### 3.2 固定顶部区域（始终展示，不可折叠）

```
┌─────────────────────────────────────┐
│  [产品图片]（如有，VIP功能）         │
│                                     │
│  产品名称                           │
│  规格：500g装                       │
│  产地：XX省XX市XX县                 │
│  生产日期：2024-03-15               │
│  生产者：XX农业有限公司              │
└─────────────────────────────────────┘
```

### 3.3 可折叠区块列表

| 区块名称 | 包含字段 | 默认状态 |
|---------|---------|---------|
| 产品详情 | 执行标准、等级、储存条件、保质期、自定义字段 | 展开 |
| 批次信息 | 批次号、采收日期、数量、产地地块、负责人 | 展开 |
| 合格证信息 | 合格证编号、开具日期、承诺声明、授权人签名 | 展开 |
| 农药使用记录 | 农药名称、登记证号、施用日期、施用量、安全间隔期 | 折叠 |
| 施肥记录 | 肥料名称、施用日期、施用量 | 折叠 |
| 生产者信息 | 企业名称、联系电话、经营地址 | 展开 |

- 区块顺序和默认展开/折叠由租户在后台配置
- 无数据的区块不展示（如该批次没有农药记录，则不显示农药记录区块）

### 3.4 合格证展示

- 合格证作为一个区块展示（一个批次可能关联多张合格证）
- 展示合格证关键信息 + 承诺声明文字
- 授权人签名图片展示
- 支持查看完整合格证PDF

## 五、强制品牌露出（B端获客入口）

### 展示规则
- 追溯页底部**强制展示**，所有租户不可关闭、不可隐藏
- 文案：**"Powered by 田迹通 · 为您的农产品开具数字合格证 →"**
- 点击跳转 web-admin 主页（手机端适配版）

### 设计原则
- 样式低调，不影响消费者追溯体验
- 对消费者无感知，对业内人士是精准引导
- 天然过滤非目标用户，只有农业从业者/采购商会点击

### 转化路径
```
业内人士扫码追溯页
       ↓
看到底部 "Powered by 田迹通" 入口
       ↓
点击跳转 web-admin 主页（手机适配）
       ↓
注册 → 完善企业信息 → 成为B端客户
```

## 六、自定义域名支持【VIP功能】

- 租户开通VIP后可绑定自定义追溯域名
- 如 `trace.客户企业.com`
- 绑定后该租户所有SKU的二维码追溯链接自动切换为自定义域名
- 系统默认域名 `trace.tianjitong.com` 同时保持有效（老二维码不失效）
- 页面顶部企业Logo展示（使用企业上传的Logo，VIP功能）

## 七bis、电子标签落地页 `/label/:code`（新增）

SKU 级扫码落地页，消费者扫外包装上的电子标签二维码进入。不依赖批次。

### 数据来源
- `GET /api/label/public/:code` — 公开接口，返回 `{sku, content, tenantName, ...}`，content 是 JSON 字符串
- 若 sku 有 `categoryTemplate`，再请求 `GET /api/category-template/:key` 拿字段定义（label + type + key）

### 渲染结构（按 `content.order` 排序）
1. **关键承诺卡片**（emerald 绿色强调块，置顶）
   - 承诺字段逐条渲染：truthy 显示 ✓ + 正常文字；falsy 显示 — + 删除线
2. **检测报告 / 附件**
   - 字段值按空白/`,、\n` 拆分为 URL 列表；图片 URL 走缩略图网格，其他链接走文件 chip
3. **营销内容**
   - `richtext` 用 `<p class="whitespace-pre-wrap">` 渲染；`multiimage` 走缩略图网格
4. **自定义卡片**
   - 由编辑器新增的 `{title, body}` 数组，按写入顺序展示

`content.order` 由管理后台拖拽保存，落地页严格遵守该顺序。

### 状态
- `status=draft` 的标签 `/api/label/public/:code` 返回 404
- 不再依赖批次状态警告（批次下架警告仅作用于 `/:traceCode`）

## 八、404 页面

访问不存在的追溯码时：

- 提示"未找到该产品追溯信息"
- 建议消费者联系生产者核实
- 不展示任何企业信息

## 九、访客模式说明

- 访客租户（`tenant_id = 1`）下的批次追溯码同样可被消费者扫到，正常展示页面，便于演示和地推材料
- 访客租户产生的扫码不计入真实统计
